## 1.unique_ptr
独占指针：独占一块内存。禁止拷贝构造和拷贝赋值，但允许移动构造和移动赋值（转移所有权）。

```cpp
template <typename T>                  // 1. 定义模板，支持管理任意类型 T 的对象
class UniquePtr {
private:
    T* ptr_ = nullptr;                 // 2. 内部唯一的裸指针，初始化为空

public:
    // 构造函数与析构函数
    explicit UniquePtr(T* ptr = nullptr) : ptr_(ptr) {} 
                                       // 3. 构造函数，用 explicit 防止隐式类型转换（如 UniquePtr<int> p = new int;）
    ~UniquePtr() { delete ptr_; }       // 4. 析构函数。UniquePtr 销毁时，自动 delete 裸指针，防止内存泄漏

    // 禁用拷贝语义（独占的核心）
    UniquePtr(const UniquePtr&) = delete; 
                                       // 5. 禁用拷贝构造：防止两个 UniquePtr 指向同一块内存
    UniquePtr& operator=(const UniquePtr&) = delete; 
                                       // 6. 禁用拷贝赋值：同上，防止通过赋值产生两个管理者

    // 允许移动语义（转移所有权）
    UniquePtr(UniquePtr&& other) noexcept : ptr_(other.ptr_) { 
                                       // 7. 移动构造：接管 other 的裸指针
        other.ptr_ = nullptr;          // 8. 将 other 的指针置空，防止 other 析构时把刚接管的内存 delete 掉
    }

    UniquePtr& operator=(UniquePtr&& other) noexcept { 
                                       // 9. 移动赋值运算符
        if (this != &other) {          // 10. 防止自我赋值（如 p = std::move(p);）
            delete ptr_;               // 11. 释放当前 UniquePtr 自己原本管理的旧内存
            ptr_ = other.ptr_;         // 12. 接管 other 的裸指针
            other.ptr_ = nullptr;      // 13. 将 other 的指针置空，断开其所有权
        }
        return *this;                  // 14. 返回自身引用，支持链式赋值
    }

    // 解引用操作符（使其用起来像普通指针）
    T& operator*() const { return *ptr_; }  // 15. 重载 * 运算符，返回裸指针所指对象的引用（如 *p）
    T* operator->() const { return ptr_; }  // 16. 重载 -> 运算符，返回裸指针本身（如 p->method()）
    T* get() const { return ptr_; }         // 17. 提供 get() 接口，方便用户获取底层的裸指针进行只读操作

    // 释放控制权
    T* release() {                     // 18. 放弃对内存的控制权，但不销毁内存
        T* temp = ptr_;                // 19. 暂存裸指针
        ptr_ = nullptr;                // 20. 内部指针置空
        return temp;                   // 21. 返回裸指针，交由外部手动管理
    }
};
```

const UniquePtr&：const 左值引用：
- &：左值引用，绑定普通左值（有名字、可重复使用的对象，如 UniquePtr p）；
- const：保证拷贝构造不会修改源对象

&& 右值引用：
- 左值：有名字、可复用对象（p1）；
- 右值：临时对象、std::move(p1) 转换后的对象，生命周期短暂；
- UniquePtr&& 只能接收右值，区分「拷贝（左值）」和「移动（右值）」。
- std::move(p)：把左值强制转为右值，触发移动语义。

noexcept：
- 标记函数不会抛出异常，给编译器优化提示：
- 容器（vector）扩容时，优先使用移动而非拷贝；
- 无异常保证，标准库优化依赖此关键字。

拷贝是「复制一份，两者共存」；移动是转移管理权，原对象放弃内存；

用于重载运算符，让自定义类像原生指针一样使用。
- operator*：解引用，*p → 返回托管对象引用；
- operator->：成员访问，p->func() → 转发底层裸指针；
- operator=：赋值运算符，区分拷贝赋值、移动赋值；

后缀 const
- 函数末尾的 const：表示该函数不会修改类的成员变量。
- get() / operator* / operator-> 都是只读接口，不会改动内部 ptr_，所以加 const，const UniquePtr 对象也能调用。

release() 函数逻辑：
- 原理：切断 UniquePtr 和堆内存的绑定，不释放内存，把裸指针返回给外部。
- 调用后 UniquePtr 内部指针为空，析构不会释放内存，内存交给调用者手动 delete，用于所有权转交第三方裸指针场景。（move）


## 2.shared_ptr
多个智能指针共同管理同一块内存。内部维护一个引用计数器，当计数器清零时，释放内存。
```cpp
template <typename T>
class SharedPtr {
private:
    T* ptr_ = nullptr;                 // 1. 指向实际对象的裸指针
    size_t* ref_count_ = nullptr;      // 2. 指向堆上引用计数的指针。必须是普通指针，才能让多个 SharedPtr 共享同一个计数器

    void release() {                   // 3. 内部私有辅助函数：用于递减计数并尝试释放内存
        if (ref_count_) {              // 4. 如果引用计数器存在
            (*ref_count_)--;           // 5. 引用计数减 1
            if (*ref_count_ == 0) {    // 6. 如果减到了 0，说明没有任何 SharedPtr 指向这块内存了
                delete ptr_;           // 7. 销毁实际的对象
                delete ref_count_;     // 8. 销毁堆上的计数器内存
            }
        }
    }

public:
    // 构造与析构
    explicit SharedPtr(T* ptr = nullptr) { 
                                       // 9. 构造函数
        if (ptr) {                     // 10. 如果传入了有效的裸指针
            ptr_ = ptr;                // 11. 保存对象指针
            ref_count_ = new size_t(1);// 12. 在堆上动态开辟一块内存来存计数器，初始值为 1
        }
    }
    ~SharedPtr() { release(); }        // 13. 析构函数：调用 release()，尝试减少计数或释放内存

    // 拷贝构造与拷贝赋值（共享的核心）
    SharedPtr(const SharedPtr& other) : ptr_(other.ptr_), ref_count_(other.ref_count_) { 
                                       // 14. 拷贝构造：直接复制对方的对象指针和计数器指针
        if (ref_count_) {              // 15. 如果对方的计数器有效
            (*ref_count_)++;           // 16. 强引用计数加 1
        }
    }

    SharedPtr& operator=(const SharedPtr& other) { 
                                       // 17. 拷贝赋值运算符
        if (this != &other) {          // 18. 防止自我赋值
            release();                 // 19. 核心！先将自己当前管理的旧资源计数减 1（如果减到0会释放旧资源）
            ptr_ = other.ptr_;         // 20. 指向新对象的指针
            ref_count_ = other.ref_count_; // 21. 指向新对象的计数器
            if (ref_count_) {          // 22. 如果新计数器有效
                (*ref_count_)++;       // 23. 新资源计数加 1
            }
        }
        return *this;                  // 24. 返回自身
    }

    // 移动构造与移动赋值（优化性能，不增减计数）
    SharedPtr(SharedPtr&& other) noexcept : ptr_(other.ptr_), ref_count_(other.ref_count_) { 
                                       // 25. 移动构造：直接接管 other 的指针和计数器
        other.ptr_ = nullptr;          // 26. 将 other 置空
        other.ref_count_ = nullptr;    // 27. 将 other 的计数器指针置空（计数不发生改变，只是管理者换了）
    }

    SharedPtr& operator=(SharedPtr&& other) noexcept { 
                                       // 28. 移动赋值
        if (this != &other) {          // 29. 防止自我赋值
            release();                 // 30. 先释放自己当前的旧资源
            ptr_ = other.ptr_;         // 31. 接管 other 的对象指针
            ref_count_ = other.ref_count_; // 32. 接管 other 的计数器指针
            other.ptr_ = nullptr;      // 33. other 置空
            other.ref_count_ = nullptr;
        }
        return *this;
    }

    // 工具函数
    T& operator*() const { return *ptr_; }   // 34. 解引用
    T* operator->() const { return ptr_; }   // 35. 箭头运算符
    size_t use_count() const { return ref_count_ ? *ref_count_ : 0; } 
                                       // 36. 获取当前有多少个 SharedPtr 共同管理该对象
    
    // 给 WeakPtr 留的后门接口
    size_t* get_ref_count() const { return ref_count_; } // 37. 允许 WeakPtr 获取计数器指针
    T* get() const { return ptr_; }                      // 38. 允许 WeakPtr 获取裸指针
};
```
## 3.weak_ptr
用来解决 SharedPtr 的循环引用问题。它指向 SharedPtr 管理的对象，但不增加其强引用计数。为了实现这一点，真实的工业代码会使用一个包含 shared_count 和 weak_count 的控制块。

```cpp
template <typename T>
class WeakPtr {
private:
    T* ptr_ = nullptr;                 // 1. 指向对象的裸指针，但不拥有所有权
    size_t* ref_count_ = nullptr;      // 2. 指向 SharedPtr 的引用计数，只看不用（不增加强引用计数）

public:
    WeakPtr() = default;               // 3. 默认构造函数

    // 从 SharedPtr 构造
    WeakPtr(const SharedPtr<T>& shared_ptr) 
        : ptr_(shared_ptr.get()), ref_count_(shared_ptr.get_ref_count()) {} 
                                       // 4. 允许通过 SharedPtr 构造。拷贝了指针，但**没有**增加 `*ref_count_` 的值

    // 拷贝与赋值
    WeakPtr(const WeakPtr& other) : ptr_(other.ptr_), ref_count_(other.ref_count_) {} 
                                       // 5. 弱指针之间的拷贝，同样不改变计数
    WeakPtr& operator=(const WeakPtr& other) { 
                                       // 6. 弱指针之间的赋值
        if (this != &other) {          // 7. 防止自我赋值
            ptr_ = other.ptr_;         // 8. 刷新内部指针
            ref_count_ = other.ref_count_;
        }
        return *this;
    }

    // 检查资源是否已被销毁
    bool expired() const {             // 9. 判断它指向的 SharedPtr 资源是否已经被释放了
        return !ref_count_ || (*ref_count_ == 0); 
                                       // 10. 如果计数器不存在，或者计数器的强引用数已经归零，说明对象已被销毁
    }

    // 提升为 SharedPtr 以便安全使用资源
    SharedPtr<T> lock() const {        // 11. 核心函数。WeakPtr 不能直接访问资源，必须升级为强引用 SharedPtr 才能用
        if (expired()) {               // 12. 如果资源已经死掉了
            return SharedPtr<T>(nullptr); // 13. 返回一个空的 SharedPtr
        }
        // 14. 简化模拟：如果资源还在，利用现有的指针，通过后门构造出一个合法的 SharedPtr
        // 注意：生产环境标准的锁操作会利用控制块（Control Block）来保证线程安全地让强引用计数加 1
        return SharedPtr<T>(*this); 
    }

    friend class SharedPtr<T>;         // 15. 声明 SharedPtr 为友元类，方便其在 lock 提升时访问私有成员
};
```

生产环境的 std::weak_ptr 会在堆上分配一个统一的 ControlBlock 控制块，即使强引用归零，只要弱引用不为零，控制块本身就不会被 delete，从而保证 WeakPtr 访问引用计数是内存安全的。


## 4.make_shared
在传统的 SharedPtr<T>(new T()) 中，程序需要执行两次独立的堆内存分配（一次给 T 对象，一次给引用计数的控制块）。而 make_shared 通过一次性分配一块连续的内存，同时容纳 T 对象和控制块。

为了完美实现 make_shared，我们需要引入一个真正的控制块（Control Block）结构，并在堆上进行整块内存的 placement new 构造。

控制块必须在堆上分配，它需要同时管理强引用计数、弱引用计数，以及负责对象本身的销毁。

```cpp
// 1. 控制块基类（用于解耦类型）
struct ControlBlockBase {
    size_t shared_count = 1;
    size_t weak_count = 0;
    
    virtual void dispose() = 0;      // 负责销毁 T 对象
    virtual void destroy() = 0;      // 负责销毁控制块自身
    virtual ~ControlBlockBase() = default;
};

// 2. 专门为 make_shared 设计的控制块（对象和计数在同一块内存中）
template <typename T>
struct MakeSharedControlBlock : public ControlBlockBase {
    // 使用 std::aligned_storage 或 char 数组来预留 T 对象的内存空间
    // 这样可以确保控制块创建时，T 对象还没有被构造
    alignas(T) char storage[sizeof(T)];

    T* get_object_ptr() {
        return reinterpret_cast<T*>(storage);
    }

    // 实现虚函数：只销毁对象，不释放内存
    void dispose() override {
        get_object_ptr()->~T(); 
    }

    // 实现虚函数：当强引用和弱引用都为 0 时，释放整块内存
    void destroy() override {
        delete this; 
    }
};

```

alignas(T) char storage[sizeof(T)] 对齐缓冲区
- sizeof(T)：数组大小刚好容纳一个完整 T 对象；
- alignas(T)：强制数组内存地址满足 T 的对齐要求；
如果不加 alignas，char 默认 1 字节对齐，placement new 构造 T 会出现未定义行为、段错误；
- 本质：一块未初始化原始内存，控制块构造时不会自动创建 T，后续靠 placement new 在 storage 上手动构造对象。



为什么要这样实现？
- 完美的异常安全：如果是 SharedPtr<T>(new T(args...))，如果在 new T 成功后，SharedPtr 的构造函数由于其他原因（如内存不足创建计数器失败）抛出异常，new T 的内存就会泄露。而 make_shared 内部是一体化构建，一旦 new MakeSharedControlBlock 失败，什么都不会分配，绝不泄露。
- CPU 缓存友好：因为对象数据和引用计数在内存上是紧挨着的，CPU 读取时会一起加载到 Cache Line 中，大幅提升了指针解引用时的性能。

### 4.1 改造后的 Shared_ptr
为了支持 make_shared，SharedPtr 内部不再维护两个分离的指针，而是直接维护一个对象指针和一个控制块指针
```cpp
template <typename T>
class SharedPtr {
private:
    T* ptr_ = nullptr;
    ControlBlockBase* ctrl_ = nullptr; // 统一通过控制块基类指针管理

    // 私有构造函数，专门留给 make_shared 调用
    SharedPtr(T* ptr, ControlBlockBase* ctrl) : ptr_(ptr), ctrl_(ctrl) {}

    void release() {
        if (ctrl_) {
            ctrl_->shared_count--;
            if (ctrl_->shared_count == 0) {
                ctrl_->dispose(); // 强引用归零，销毁 T 对象
                if (ctrl_->weak_count == 0) {
                    ctrl_->destroy(); // 弱引用也为 0，销毁整个控制块内存
                }
            }
        }
    }

public:
    SharedPtr() = default;
    
    // 传统的常规构造函数（依旧保留两次内存分配的语义）
    explicit SharedPtr(T* ptr) {
        if (ptr) {
            ptr_ = ptr;
            // 普通构造需要一个单独的控制块，去 delete 外部传进来的 ptr
            // 篇幅原因此处略去常规控制块子类的定义，核心看 make_shared
        }
    }

    ~SharedPtr() { release(); }

    // 拷贝构造（增加强引用计数）
    SharedPtr(const SharedPtr& other) : ptr_(other.ptr_), ctrl_(other.ctrl_) {
        if (ctrl_) ctrl_->shared_count++;
    }

    // 移动构造
    SharedPtr(SharedPtr&& other) noexcept : ptr_(other.ptr_), ctrl_(other.ctrl_) {
        other.ptr_ = nullptr;
        other.ctrl_ = nullptr;
    }

    // 声明为友元，允许 make_shared 访问私有构造函数
    template <typename U, typename... Args>
    friend SharedPtr<U> make_shared(Args&&... args);

    // 操作符重载
    T& operator*() const { return *ptr_; }
    T* operator->() const { return ptr_; }
    size_t use_count() const { return ctrl_ ? ctrl_->shared_count : 0; }
};
```


关键设计思想（为什么要拆分 dispose /destroy）
1. 对象生命周期 和 控制块内存生命周期分离
    - 对象 T：由 shared_count 控制，无 SharedPtr 就销毁；
    - 控制块：由 shared_count + weak_count 共同控制，两者都为 0 才释放；
2. 适配 WeakPtr 弱指针场景
    - WeakPtr 不持有对象所有权，但需要读取计数判断对象存活，因此对象销毁后控制块必须暂时保留；
3. make_shared 合并内存约束
    - make_shared 的对象和计数在同一块连续堆内存，不能单独 delete T；只能先析构对象（dispose），等无任何指针后一次性 delete 整块内存（destroy）；
4. 分层释放避免内存泄漏 / 段错误
    - 如果不分两步，直接在 shared_count==0 就 delete this，WeakPtr 后续访问空控制块会直接崩溃。
