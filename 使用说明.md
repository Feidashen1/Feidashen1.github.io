## 1. 快速开始：发一篇文章

三步发布一篇新文章：

```bash
cd /home/fei/opensource/github/Feidashen1.github.io

# 1) 新建文章
hugo new posts/我的标题.md

# 2) 编辑内容（用任何编辑器打开 content/posts/我的标题.md）
#    把 draft 那行删掉，或改成 draft: false

# 3) 发布
git add .
git commit -m "新文章：我的标题"
git push
```

push 后 1~3 分钟，文章自动出现在 https://feidashen1.github.io 。

> 想本地先看看效果？运行 `hugo server`，浏览器打开 http://localhost:1313 ，
> 边改边自动刷新，满意了再 push。

## 2. 项目结构

```
Feidashen1.github.io/
├── hugo.toml               # ⭐ 站点总配置（标题、菜单、主题参数）
├── content/
│   ├── posts/              # ⭐ 你的所有文章都放这里
│   │   └── first-post.md
│   ├── search.md           # 搜索页（不用动）
│   └── archives.md         # 归档页（不用动）
├── static/                 # 放图片等静态文件，URL 直接 /图片名
├── themes/PaperMod/        # 主题（git submodule，一般不直接改）
├── archetypes/
│   └── default.md          # 新建文章的模板
├── .github/workflows/
│   └── hugo.yml            # 自动部署配置（不用动）
└── 使用说明.md             # 本文件
```

**你日常只需要碰两个地方**：`content/posts/`（写文章）和 `hugo.toml`（改配置）。

## 3. 写文章详解

### 新建文章

```bash
hugo new posts/文章文件名.md
```

文件名建议用英文或拼音（影响网址），中文标题写在文件里的 `title` 字段。
例如 `hugo new posts/hello-hugo.md`，网址就是 `/posts/hello-hugo/`。

### 文章正文用 Markdown

````markdown
## 二级标题

正文段落，支持 **加粗**、*斜体*、`行内代码`、[链接](https://example.com)。

> 引用块

- 无序列表
1. 有序列表

```python
print("代码块，带语法高亮和复制按钮")
```

![图片说明](/my-image.png)   ← 图片先放到 static/ 目录
````

### 插入图片

1. 把图片放进 `static/` 目录，比如 `static/photo.jpg`
2. 文章里引用：`![说明](/photo.jpg)`（注意开头的 `/`）

### 「阅读更多」摘要分隔

在文章中插入 `<!--more-->`，首页列表只显示这行之前的内容作为摘要。

## 4. Front Matter（文章头部）字段

每篇文章顶部 `---` 之间的部分，控制文章属性：

```yaml
---
title: "文章标题"              # 显示的标题
date: 2026-06-02T14:00:00+08:00  # 发布时间
draft: false                   # true=草稿不发布，false=正式发布
tags: ["Hugo", "教程"]         # 标签（点击可聚合）
categories: ["技术"]           # 分类
summary: "一句话摘要"          # 列表/SEO 显示的摘要（可选）
weight: 1                      # 置顶用，数字越小越靠前（可选）
ShowToc: true                  # 是否显示本文目录（可选）
---
```

**最关键的是 `draft`**：写作时设 `true` 不会上线，写完改成 `false`（或删掉）才会发布。

## 6. 发布上线

本博客用 **GitHub Actions 自动部署**，你只管 push 源码：

```bash
git add .
git commit -m "描述这次改了啥"
git push
```

push 后：
- 去仓库的 **Actions** 标签页能看到部署进度（绿勾=成功）
- 通常 1~3 分钟后线上更新

> 推送时若提示输入账号密码：Username 填 `Feidashen1`，Password 填你的
> **个人访问令牌（ghp_... 那串）**，不是账号密码。令牌需要勾选 `repo` + `workflow` 权限。

## 7. 常用定制

所有站点配置都在 **`hugo.toml`**，改完 `git push` 即可生效。

### 改博客标题 / 首页问候语

```toml
title = "你的新标题"

[params.homeInfoParams]
  Title = "👋 你好，我是 XXX"
  Content = "这里写你的自我介绍。"
```

### 改导航菜单

在 `[menu]` 块里增删 `[[menu.main]]`，`weight` 控制顺序。

### 改深色/浅色模式默认值

```toml
[params]
  defaultTheme = "auto"   # auto=跟随系统 / light=亮 / dark=暗
```

### 增删社交图标

```toml
[[params.socialIcons]]
  name = "github"          # 支持 github/twitter/email/zhihu/bilibili 等
  url = "https://github.com/Feidashen1"
```

### 加头像（首页/侧边）

把头像图片放 `static/`，然后：

```toml
[params.homeInfoParams]
  # ...
# 或者用 profile 模式，参考 PaperMod 文档
```

更多参数见 PaperMod 官方 wiki：
https://github.com/adityatelange/hugo-PaperMod/wiki

## 9. 常见问题排查

| 现象 | 原因 / 解决 |
|------|------|
| push 报 `refusing to allow a Personal Access Token ... workflow` | 令牌缺 `workflow` 权限。去 https://github.com/settings/tokens 编辑令牌勾上 `workflow`，Update。 |
| push 提示 `Password authentication is not supported` | 别用账号密码，用个人访问令牌（PAT）当密码。 |
| 文章 push 了但线上没出现 | 检查 `draft` 是不是 `true`；去 Actions 看部署是否成功。 |
| Actions 报错找不到主题 | submodule 没拉取，确认 `.gitmodules` 已提交、workflow 里有 `submodules: recursive`（本项目已配好）。 |
| 本地 `hugo` 命令找不到 | Hugo 装在 `~/.local/bin/hugo`，确认它在 PATH 里，或用全路径。 |
| 网址打不开（404） | 确认仓库名是 `Feidashen1.github.io`，且 Settings→Pages→Source 选的是 **GitHub Actions**。 |

查看部署日志：仓库 → **Actions** 标签页 → 点最近一次运行。

## 10. 换电脑/重装后如何继续

```bash
# 1) 装 Hugo extended（版本见 .github/workflows/hugo.yml 里的 HUGO_VERSION）
#    从 https://github.com/gohugoio/hugo/releases 下 hugo_extended_*_linux-amd64.tar.gz

# 2) 克隆仓库（注意 --recurse-submodules 把主题一起拉下来）
git clone --recurse-submodules git@github.com:Feidashen1/Feidashen1.github.io.git
# 或 HTTPS：
git clone --recurse-submodules https://github.com/Feidashen1/Feidashen1.github.io.git

# 3) 进目录，正常使用
cd Feidashen1.github.io
hugo server
```

> 如果克隆时忘了 `--recurse-submodules`，补一句：
> `git submodule update --init --recursive`

## 11. Makefile 快捷命令

项目根目录有 `Makefile`，把常用操作缩短成一条命令：

```bash
make help                  # 查看所有命令
make new t=我的标题        # 新建文章 content/posts/我的标题.md
make serve                 # 本地预览（含草稿）http://localhost:1313
make build                 # 本地构建检查
make publish m="更新说明"  # add + commit + push 一键发布
make upgrade               # 升级 PaperMod 主题
make clean                 # 清理构建产物
```

> 没装 `make` 的话：`sudo apt install make`。也可继续用原始 hugo/git 命令。

## 12. 开启评论（Giscus）

Giscus 基于 GitHub Discussions，免费、无需后端，适合技术博客。

1. 仓库 **Settings → General → Features** 勾选 **Discussions** 开启讨论区。
2. 安装 giscus app：打开 https://github.com/apps/giscus → Install，授权给
   `Feidashen1.github.io` 仓库。
3. 打开 https://giscus.app/zh-CN ，"仓库"填 `Feidashen1/Feidashen1.github.io`，
   页面会自动生成 **repo-id** 和 **category-id**；分类建议选 `Announcements`。
4. 把四个值填进 `hugo.toml` 的 `[params.giscus]`：

   ```toml
   [params.giscus]
     repo = "Feidashen1/Feidashen1.github.io"
     repoId = "R_xxxxxxxx"
     category = "Announcements"
     categoryId = "DIC_xxxxxxxx"
   ```

5. `make publish m="开启评论"`。文章底部即出现评论框，读者用 GitHub 账号即可评论。

> `repo` 留空不显示评论，不影响站点。单篇关闭评论：该文 front matter 加 `comments: false`。

## 13. 开启访问统计（Google Analytics）

1. 打开 https://analytics.google.com 创建"媒体资源"，数据流选 **网站**，
   填 `https://feidashen1.github.io`，拿到测量 ID `G-XXXXXXXXXX`。
2. 填进 `hugo.toml` 末尾：

   ```toml
   [services.googleAnalytics]
     ID = "G-XXXXXXXXXX"
   ```

3. `make publish m="接入 GA"`。

> 统计代码**只在线上加载**，本地 `hugo server` 预览不会污染数据；ID 留空则完全不加载。
> 想要更隐私友好的方案，可换 [Umami](https://umami.is/) 或百度统计。
